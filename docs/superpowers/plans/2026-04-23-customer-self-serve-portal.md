# Customer Self-Serve Portal Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a customer-facing self-serve portal that allows customers to confirm appointments, pay invoices, and request reschedules via tokenized links — no account required. Tokens are single-use (except reschedule), expire in 72 hours, and are served via public routes mounted before Clerk authentication middleware.

**Architecture:** A new `portal_tokens` table stores short-lived UUID tokens scoped to a tenant, purpose, and entity. Public `/portal/*` Express routes validate tokens and delegate to existing services (Stripe, appointment/invoice repos). The React frontend renders three portal pages using the token from the URL, with `InvoicePaymentPage.tsx` wired to the real API for the first time.

**Tech Stack:** TypeScript, Express, `pg` driver, Zod for validation, React + Tailwind on the frontend. Vitest for all tests (InMemory repo pattern for unit, supertest for integration).

### Task 3: PortalTokenService — creation and validation logic

import { InMemoryPortalTokenRepo } from '../src/db/repos/portalTokenRepo';

describe('PortalTokenService', () => {
  let svc: PortalTokenService;
  let repo: InMemoryPortalTokenRepo;

    repo = new InMemoryPortalTokenRepo();
    svc = new PortalTokenService(repo);

type ErrorCode = 'NOT_FOUND' | 'EXPIRED' | 'USED' | 'UNKNOWN';

  NOT_FOUND: { title: 'Link not found', body: 'This link does not exist. Please contact us for help.' },
  EXPIRED:   { title: 'Link expired', body: 'This link expired after 72 hours. Please contact us to get a new one.' },
  USED:      { title: 'Already used', body: 'This link has already been used.' },
  UNKNOWN:   { title: 'Something went wrong', body: 'Please contact support.' },
};

export function PortalError({ code }: Props) {
  const { title, body } = messages[code];
  return (
    <div className="min-h-screen flex items-center justify-center bg-gray-50 p-6">
      <div className="max-w-md w-full bg-white rounded-xl shadow p-8 text-center space-y-4">
        <h1 className="text-2xl font-semibold text-gray-800">{title}</h1>
        <p className="text-gray-600">{body}</p>
      </div>
    </div>
  );
}

## Out of scope

- Twilio SMS delivery — `SmsReminderDispatcher` is a shell only; implementation is covered by a separate Twilio SMS delivery plan
- Stripe integration changes — the portal pay route delegates to the existing Stripe payment intent service without modification
- Customer account creation or authentication — the portal is intentionally account-free
- Email delivery of portal links — links are returned to the caller via API response only; email delivery is a separate concern
- Portal link revocation — there is no admin UI to invalidate a token before it expires; expiry window is the only mechanism
- Push notifications or webhooks on confirmation/reschedule — out of scope for this plan
- Pagination or history of past confirmations/reschedule requests in the customer portal
- Analytics or tracking pixels on portal page loads
- Custom branding per tenant on portal pages (white-labelling is a separate plan)
